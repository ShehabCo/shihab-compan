## نظرة عامة على المشروع

**منصة شهاب** هي سوق رقمي عربي متكامل يربط بين مقدمي الخدمات الرقمية والعملاء. المنصة مصممة بالكامل باللغة العربية مع دعم كامل لاتجاه النص من اليمين إلى اليسار (RTL).

### المميزات الرئيسية

#### 1. نظام المستخدمين متعدد الأدوار
- **مشتري**: يمكنه تصفح الخدمات، الشراء، والتواصل مع البائعين
- **بائع**: يمكنه إضافة خدمات، إدارة الطلبات، وسحب الأرباح
- **مدير**: لديه صلاحيات كاملة لإدارة المنصة

#### 2. السوق الرقمي
- تصفح الخدمات حسب الفئات (تصميم، برمجة، كتابة، تسويق، إلخ)
- نظام بحث متقدم
- فلترة حسب السعر والتقييم
- صفحات تفصيلية لكل خدمة مع معلومات البائع

#### 3. لوحة تحكم البائع
- إضافة وتعديل الخدمات
- متابعة الطلبات والإيرادات
- إدارة المحفظة والسحوبات
- عرض التقييمات والمراجعات

#### 4. نظام المحفظة والمدفوعات
- رصيد متاح ورصيد معلق
- سجل المعاملات الكامل
- طلبات السحب
- عمولة المنصة 10% تلقائياً

#### 5. نظام المراسلة
- محادثة مباشرة بين المشتري والبائع
- ربط المحادثات بالطلبات
- واجهة عربية كاملة RTL

#### 6. إدارة الطلبات
- تتبع حالة الطلب (جديد، قيد التنفيذ، مُسلّم، مكتمل)
- تسليم الملفات
- قبول أو رفض التسليم
- تحديث تلقائي للرصيد عند الإكمال

#### 7. نظام التقييمات
- تقييم بالنجوم من 1 إلى 5
- تعليقات اختيارية
- عرض التقييمات على صفحة الخدمة
- إحصائيات شاملة للبائعين

#### 8. لوحة تحكم المدير
- إحصائيات شاملة للمنصة
- إدارة المستخدمين (تغيير الأدوار، الحظر)
- إدارة الخدمات (الموافقة، الحذف)
- متابعة الطلبات والمعاملات المالية

## التقنيات المستخدمة

### Frontend
- **Next.js 16** - إطار عمل React الحديث
- **React 19** - مكتبة واجهة المستخدم
- **TypeScript** - للكود الآمن والموثوق
- **Tailwind CSS v4** - تصميم سريع ومرن
- **shadcn/ui** - مكونات واجهة مستخدم جاهزة

### Backend
- **Supabase** - قاعدة بيانات PostgreSQL
- **Supabase Auth** - نظام المصادقة
- **Row Level Security (RLS)** - حماية البيانات

### التصميم
- **خط Cairo** - مُحسّن للغة العربية
- **نظام ألوان مخصص**:
  - خلفية داكنة (navy)
  - بطاقات بيضاء نظيفة
  - لون مميز (coral/orange)
- **دعم كامل لـ RTL** - تجربة عربية أصيلة

## هيكل قاعدة البيانات

### الجداول الرئيسية

1. **profiles** - ملفات المستخدمين
   - الاسم، الصورة، الدور، السيرة الذاتية
   - التقييم العام وعدد الخدمات المكتملة

2. **services** - الخدمات المعروضة
   - العنوان، الوصف، الفئة، السعر
   - مدة التسليم، الحالة (نشط/معطل)
   - ارتباط بالبائع

3. **orders** - الطلبات
   - ربط بين المشتري والبائع والخدمة
   - حالة الطلب، المتطلبات، ملفات التسليم
   - السعر الإجمالي، عمولة المنصة

4. **wallets** - المحافظ المالية
   - الرصيد المتاح، الرصيد المعلق
   - إجمالي الأرباح والسحوبات
   - محفظة لكل بائع

5. **transactions** - المعاملات المالية
   - نوع المعاملة (إضافة، سحب، عمولة)
   - المبلغ، الحالة، الوصف

6. **reviews** - التقييمات
   - التقييم (1-5 نجوم)
   - التعليق، ربط بالطلب والخدمة

7. **messages** - الرسائل
   - محادثات بين المشتري والبائع
   - ربط مع الطلب

## الأمان والحماية

### Row Level Security (RLS)
جميع الجداول محمية بسياسات RLS:
- المستخدمون يرون بياناتهم فقط
- البائعون يديرون خدماتهم فقط
- المدراء لديهم صلاحيات كاملة

### المصادقة
- تسجيل الدخول عبر البريد الإلكتروني
- التحقق من البريد الإلكتروني
- حماية الجلسات
- Middleware للمسارات المحمية

## دليل النشر السريع

### الخطوة 1: إعداد Supabase

1. قم بتسجيل الدخول إلى [Supabase](https://supabase.com)
2. قم بتشغيل جميع السكريبتات في مجلد `scripts/` بالترتيب:
   - `001_create_users_and_profiles.sql`
   - `002_create_services.sql`
   - `003_create_orders.sql`
   - `004_create_wallets.sql`
   - `005_create_reviews.sql`
   - `006_create_messages.sql`
   - `007_seed_data.sql` (بيانات تجريبية)

### الخطوة 2: النشر على Vercel

1. انقر على زر "Publish" في أعلى الشاشة
2. سيتم ربط المشروع تلقائياً بـ Supabase
3. جميع متغيرات البيئة موجودة بالفعل

### الخطوة 3: الوصول إلى المنصة

بعد النشر ستحصل على رابط مثل:
```
https://your-project.vercel.app
```
