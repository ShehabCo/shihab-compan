TROUBLESHOOTING
================================================================================

ERROR: "Module not found: Can't resolve '@/lib/supabase/proxy'"
SOLUTION: rm -rf .next && pnpm install && pnpm dev

ERROR: "GROQ_API_KEY is not defined"
SOLUTION: Check .env.local has GROQ_API_KEY value

ERROR: "Cannot connect to Supabase"
SOLUTION: Check NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY

ERROR: "Stripe is not configured"
SOLUTION: Check NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY is present

ERROR: Database shows no tables
SOLUTION: Run /scripts/001_create_tables.sql in Supabase SQL editor

WHAT HAPPENS WHEN YOU RUN pnpm dev?
================================================================================

1. Next.js starts dev server on localhost:3000
2. Connects to Supabase (database checks)
3. Loads Groq API key (AI ready)
4. Loads Stripe keys (payments ready)
5. Hot Module Replacement enabled (edit files, see changes instantly)

If any integration is missing, you'll see an error. That's good - it means
validation is working.
